
---

**`TDD`** - Test-Driven Development

**`Unit Testing`** - involves testing individual components in an application

**`User Interface Testing`** - involves testing the user interface components on different devices or in different browsers

**`Integration Testing`** - involves testing the compatibility of multiple components in an application

**`Functional Testing`** - involves testing the functionality of an application
- Are the customers needs satisfied?

**`Performance Testing`** - involves testing the performance of an application
- Does the application perform well?

**`Security Testing`** - involves testing the security of an application

---

**Unit Testing** is the testing of individual software components in isolation from the rest of a system.
